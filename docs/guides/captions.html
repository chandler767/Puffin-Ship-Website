Captions Guide

Puffin Ship generates per-word highlighted captions — each word lights up in highlight_color when it's being spoken.

Setup

You need a finalized transcript and a backbone cut. Run captions after your edit is stable (changing the cut after generating captions requires re-running captions).

- id: captions
  processor: captions
  input:
    - id: transcript
      step: correct_transcript   # use the corrected transcript
    - id: backbone
      step: ai_cut               # maps source timestamps to timeline positions
  params:
    max_words: 5
    font_size: 72
    position_y: -480
    highlight_color: "red"
  output:
    timeline: captions
  cache: true

The manifest

On the first run, captions writes a manifest file to your project directory:

<project>/
└── captions.yaml   # (step ID + ".yaml")

This manifest controls everything about the captions output. Subsequent runs read from the manifest rather than recomputing from the transcript.

Manifest structure

- start: 0.12
  end: 0.82
  text: "Hello world"
- start: 1.10
  end: 1.80
  text: "this is a caption"

Each entry is one caption group — displayed together on screen as a single rendered clip.

Editing the manifest

Fix groupings: Merge two entries by combining their text and adjusting start/end. Split an entry by duplicating it and dividing the time range.

Fix text: Change the text value directly. The timing stays the same.

Remove a caption: Delete the entry from the YAML list.

Add punctuation: Edit text: "Hello world" to text: "Hello, world." — edit the string freely.

After editing, re-run puffinship run — the captions step reads the updated manifest and regenerates the affected clips.

Regenerating from scratch

Delete the manifest file:

rm <project>/captions.yaml

Then re-run. A fresh manifest is generated from the transcript.

word_corrections

Use word_corrections to fix common TTS mishears before grouping. These are applied before the manifest is generated:

params:
  word_corrections:
    "ninety nine percent": "99%"
    "red panda": "Redpanda"
    "cue ell": "SQL"

Multi-word corrections merge the matched words into a single manifest entry with redistributed timestamps — the correction doesn't break timing.

Positioning

position_y is in FCP pixels from center. Negative values are below center:

  • -480 — near the bottom of a 1080×1920 frame (good default for portrait)
  • -200 — lower-third area
  • 0 — center of frame

scale adjusts the overall size of the caption group relative to font_size.
