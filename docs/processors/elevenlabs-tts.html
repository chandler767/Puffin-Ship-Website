elevenlabs_tts

Generates audio from text using the ElevenLabs text-to-speech API.

When to use

Use to generate high-quality voice-over audio from a script. The generated MP3 is saved to .puffinship/elevenlabs/<step_id>.mp3.

Important: elevenlabs_tts outputs Analysis (a file reference), not EditDecisions. It does NOT place itself on the timeline. You must follow it with a separate audio step to place the MP3 on a timeline level, and a separate generate_transcript step pointing to the file directly.

YAML Example — complete voiceover pipeline

# 1. Generate the voiceover
- id: voiceover
  processor: elevenlabs_tts
  input:
    - id: source
      step: prep_script       # must be a transcript_aiprep step (outputs Analysis.Source)
  params:
    voice_id: "your-voice-id"
    model: eleven_v3
    stability: 0.5
    similarity_boost: 0.75
    speed: 1.0
  cache: true

# 2. Place the MP3 on the audio backbone level
#    (elevenlabs_tts does NOT do this itself — audio step required)
- id: place_vo
  processor: audio
  input:
    - id: source
      file: ".puffinship/elevenlabs/voiceover.mp3"   # <-- step_id is "voiceover"
  cache_deps:
    - voiceover
  params:
    loop: false
    auto_duration: true
    gain_db: 0
  output:
    timeline: vo_audio       # must be a backbone: true, type: audio level
  cache: true

# 3. Transcribe for captions — also references the file directly
- id: transcribe_vo
  processor: generate_transcript
  input:
    - id: source
      file: ".puffinship/elevenlabs/voiceover.mp3"
  cache_deps:
    - voiceover
  params:
    language: "en"
  cache: true

Timeline level required

The timeline must declare an audio backbone level for place_vo to target:

timeline:
  levels:
    - name: vo_audio
      order: 1
      type: audio
      backbone: true
    - name: main
      order: 2
      type: video
      backbone: false

Required secrets

ELEVENLABS_API_KEY — set in workspace or template secrets. A run whose pipeline includes this processor is refused before it starts if the key is missing; the web UI prompts for it.

Inputs

ID Source Description
source step or file Text content to speak. Step input must be a transcript_aiprep output (uses Analysis.Source); file input reads the text file directly. Do NOT pass a write_script step directly — its output type is Script, not Analysis.

Params

Param Type Default Description
voice_id string — ElevenLabs voice ID
model select: eleven_v3, eleven_multilingual_v2, eleven_turbo_v2_5 "eleven_v3" ElevenLabs model ID
output_format select: mp3_44100_128, mp3_44100_192, pcm_16000, pcm_22050, pcm_44100 "mp3_44100_128" Audio output format
stability number — Voice stability (0–1). Higher = more consistent, less expressive.
similarity_boost number — Similarity to original voice (0–1)
style number — Style exaggeration (0–1)
speed number 1 Speaking speed multiplier
speaker_boost bool — Enhance speaker clarity

Output

Analysis with Source pointing to .puffinship/elevenlabs/<step_id>.mp3. This is not an EditDecision — it does not go on the timeline by itself. Use a follow-up audio step with file: ".puffinship/elevenlabs/<step_id>.mp3" and cache_deps: [<step_id>] to place it.

Notes

  • Requires PUFFINSHIP_ELEVENLABS_API_KEY.
  • Always run transcript_aiprep with target: "elevenlabs" before this step.
  • Use cache: true — generation is deterministic for the same text and params.
  • generate_transcript on the voiceover output must also use file: ".puffinship/elevenlabs/<step_id>.mp3" with cache_deps — it cannot use step: voiceover because the step output is Analysis, not a file path that generate_transcript accepts.
