stitch

Places multiple files end-to-end on a timeline level.

When to use

Use stitch when you have several pre-cut source files that should play back-to-back. For trimming before stitching, run trim steps first and reference their step outputs.

YAML Example

- id: concat_clips
  processor: stitch
  input:
    - id: sources
      files:
        - footage-a
        - footage-b
        - footage-c
  output:
    timeline: main

Inputs

ID Source Description
sources files (list) Two or more source files to concatenate, in order

Params

This processor has no configurable parameters.

Output

EditDecisions — one edit decision per source file, placed sequentially.

Notes

  • Files are placed in the order listed under files:.
  • Each file's full duration is used. Run trim on individual files first if you need to cut them before stitching.
