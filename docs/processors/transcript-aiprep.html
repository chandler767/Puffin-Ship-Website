transcript_aiprep

Prepares transcript text for TTS synthesis using Claude. Fixes grammar, expands acronyms, converts numbers and symbols to spoken form, and adds phonetic hints for ambiguous terms.

When to use

Run before elevenlabs_tts or heygen_speaker when the source transcript contains raw text that would sound unnatural when read aloud (e.g. 99%, $10M, CQRS, kubectl).

YAML Example

# When source is a static file in the workspace:
- id: prep_script
  processor: transcript_aiprep
  input:
    - id: source
      file: script-text          # must be a file input — step: references are NOT accepted
  params:
    target: "elevenlabs"
  cache: true

Chaining from write_script

transcript_aiprep does not accept step inputs. When chaining from a write_script step, reference its output file directly:

- id: write_script
  processor: write_script
  ...
  cache: true

- id: prep_script
  processor: transcript_aiprep
  input:
    - id: source
      file: ".puffinship/write_script/write_script.txt"  # step_id = "write_script"
  cache_deps:
    - write_script    # ensures write_script runs first and cache invalidates correctly
  params:
    target: "elevenlabs"
  cache: true

Inputs

ID Source Description
source file only Transcript or script text file. Must be a file: input — step: references are not accepted and will fail validation. To consume write_script output, use file: ".puffinship/write_script/<step_id>.txt" with cache_deps.

Params

Param Type Default Description
target select: heygen, elevenlabs "heygen" TTS target: "heygen" or "elevenlabs". Adjusts the cleaning rules for each system's quirks.

Output

Analysis containing cleaned script text ready for TTS.

Notes

  • Requires PUFFINSHIP_ANTHROPIC_API_KEY.
  • The cleaning rules include: grammar fixes, brand name preservation (Redpanda → Redpanda, not "red panda"), acronym expansion where needed, numbers to words (10M → "ten million"), symbols to words (99% → "ninety-nine percent"), phonetic spelling for technical terms that TTS mispronounces.
  • Use cache: true — the output is deterministic for the same transcript.
