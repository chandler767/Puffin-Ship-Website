Data Model

RationalTime

Time in Puffin Ship is always a rational number — a numerator and denominator — not a float. This maps directly to FCPXML timecodes and avoids floating-point drift across long timelines.

type RationalTime struct {
    Num int64
    Den int64
}

Important: always use schema.Zero for a zero time value. Never use schema.RationalTime{} — a zero denominator causes a panic.

Common operations: Add, Sub, Mul, ToSeconds (methods); schema.FromSeconds(sec float64) (package function).

EditDecision

The unit of editing. Every clip placed on the timeline is an EditDecision.

type EditDecision struct {
    ID         string
    Source     string         // source file path (empty for title type)
    Type       string         // "video" | "audio" | "image" | "title"
    In         *RationalTime  // nil = start of source
    Out        *RationalTime  // nil = end of source
    Placement  Placement      // level + position on the timeline
    Transforms Transforms     // scale, position, volume, etc.
    Effects    []Effect       // filters (chroma key, etc.)
    TitleCard  *TitleCard     // rendered text overlay (for title_card processor)
}

type Placement struct {
    Level    string       // timeline level (e.g. "main", "broll")
    Position RationalTime // timeline position (set by the runner)
    Anchor   *AnchorRef   // optional anchor to backbone transcript position
}

ProcessorOutput

A union type — a processor returns exactly one of these variants:

Variant Used by
EditDecisions Most processors — produces clips on the timeline
Analysis generate_transcript, workspace_analyze, analyze_footage, elevenlabs_tts, transcript_aiprep, assemble_transcript — produces data consumed by later steps
Script write_script — plain text narration script
SegmentPlan video_director — assigns content types to time ranges
TransformPatches reframe, motion_center, speaker_center, scale_transform, volume_adjust — adjusts transforms on existing edit decisions
SourceRemaps chroma_key, noise_reduce — substitutes source file paths

AnalysisResult

The Analysis variant of ProcessorOutput. Contains:

  • Transcript — word-level timestamps (from Whisper or assembled from per-clip transcripts)
  • FootageAnalysis — per-clip quality score, key moments, visual summary (from analyze_footage)
  • Source — path to a generated file (from elevenlabs_tts, etc.)

SegmentPlan

Output of the video_director processor. Assigns content types to time ranges so downstream processors know what kind of content to place where.

type SegmentPlan struct {
    Segments     []SegmentAssignment
    TotalSec     float64  // total timeline duration
    DirectorNote string   // optional note from the director
}

type SegmentAssignment struct {
    StartSec    float64  // seconds
    EndSec      float64  // seconds
    ContentType string   // e.g. "speaker", "drawings", "broll", "empty"
    Reason      string   // optional rationale
    Tags        []string // optional tags
}

Processors like broll_place, broll_imagen_generate, drawings_ai_placer, and broll_screenshot_websource check the segment plan before placing clips.
