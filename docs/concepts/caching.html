Caching

How it works

Every step with cache: true saves its output to .puffinship/cache/<stepID>.json (with the hash recorded alongside in <stepID>.meta.json). On the next run, if the hash matches, the step is skipped and the cached output is returned immediately.

The cache key is:

hash = CombineHashes(inputHash, configHash)
  • inputHash — hash of all input file contents and previous step output hashes
  • configHash — hash of the step's params and prompt

When a step re-runs

A cached step re-runs when:

  • Any input file has changed (content hash changed)
  • Any upstream step's output has changed
  • The step's params or prompt has changed
  • A step listed under cache_deps has changed (even if not used as direct input)

cache_deps

Use cache_deps to invalidate a step when an upstream step changes, even if you don't use its output directly:

- id: background
  processor: fill_level
  input:
    - id: source
      file: background-image
  cache_deps:
    - ai_cut           # re-run when ai_cut changes, even though we don't read its output
  params:
    duration_step: ai_cut
  cache: true

Without cache_deps, the background step would cache forever even if ai_cut produces a shorter timeline.

Clearing cache

Cache is cleared from the template view in the web app, which calls:

DELETE /api/workspaces/{id}/templates/{tid}/cache              # clear every step
DELETE /api/workspaces/{id}/templates/{tid}/cache/from/{stepId} # clear from one step onward

Clearing from a step also invalidates every step after it, since their inputs have changed.

Manifest caches

Some processors maintain their own manifest files separate from the step cache:

Processor Manifest location Contains
broll_place / broll_veo_generate broll/manifest.json (project root) Per-clip analysis, verdict, trim points
broll_imagen_generate broll-imagen/manifest.json (project root) Per-image generation record
heygen_speaker heygen/manifest.json (project root, or <output_dir>/) Per-clip generation record
drawings_ai_placer broll-drawings/manifest.json (project root) Per-drawing placement record

These manifests persist across runs independently of the step cache. To force re-generation of a specific clip, delete its entry from the manifest (or delete the whole file to regenerate everything).
