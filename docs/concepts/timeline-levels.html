Timeline & Levels

What is a level?

A level is a named track in the timeline. Every EditDecision belongs to a level. Levels are declared in the timeline.levels section of your config:

timeline:
  levels:
    - name: music
      order: -1
      type: audio
      backbone: false
    - name: main
      order: 1
      type: any
      backbone: true
    - name: broll
      order: 2
      type: video
      backbone: false
    - name: captions
      order: 4
      type: video
      backbone: false

The backbone level

Exactly one level must be backbone: true. The backbone level:

  • Drives total timeline duration
  • Maps directly to the FCP <spine> element
  • Is the reference point for all other levels

All non-backbone clips are positioned relative to backbone time.

Non-backbone levels and lanes

Non-backbone levels map to FCP lane attributes. The lane number is computed as:

lane = levelOrder - backboneOrder

So with backbone at order 1: broll (order 2) is lane 1, captions (order 4) is lane 3, music (order -1) is lane -2.

Lower order numbers render further back in the canvas.

Multiple backbone steps: intersection

If two or more steps output to the backbone level, their edit decisions are intersected — only segments that both steps agree to keep survive. This is used when you have multiple independent AI cuts that must all agree on what to include.

Non-backbone steps: union

Multiple steps writing to the same non-backbone level are unioned — all clips are kept. This is how broll_place and broll_veo_generate both write to the broll level without conflicting.

Level types

Type Use
video Video-only clips
audio Audio-only clips
any Mixed video/audio
title FCP title clips

Querying levels from a processor

Processors can read the full state of any level at any point during pipeline execution:

ctx.AllEditDecisionsAtLevel("broll")

This is used by broll_place and broll_veo_generate to check what's already placed before deciding where to put new clips.
